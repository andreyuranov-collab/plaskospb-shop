# Инструкция: загрузка PLASKO SPB через GitHub Upload и Cloudflare Pages

Эта инструкция рассчитана на простой сценарий, как с сайтом ЛП: без командной строки, без `git init`, без `git push`.

## 1. Подготовить файлы

1. Скачать и распаковать архив `plaskospb_shop_v00_01_test_260707.zip`.
2. Открыть распакованную папку.
3. Убедиться, что в корне папки лежат файлы и папки:
   - `index.html`
   - `styles.css`
   - `script.js`
   - `assets/`
   - `data/`
   - `docs/`
   - `_headers`
   - `robots.txt`
   - `sitemap.xml`
   - `README.md`
   - `CHANGELOG.md`

Важно: на GitHub нужно загружать **содержимое папки**, а не саму папку целиком внутрь репозитория. То есть `index.html` должен лежать сразу в корне репозитория.

## 2. Создать репозиторий на GitHub

1. Открыть GitHub.
2. Нажать **New repository**.
3. Название, например:

```text
plaskospb-shop
```

4. Public/Private — на выбор.
5. README можно не создавать, потому что файл `README.md` уже есть в архиве.
6. Нажать **Create repository**.

## 3. Загрузить файлы через веб-интерфейс GitHub

1. В новом пустом репозитории нажать **uploading an existing file** или **Add file → Upload files**.
2. Перетащить в окно загрузки **всё содержимое** распакованной папки:
   - файлы из корня;
   - папки `assets`, `data`, `docs`.
3. В поле commit message написать, например:

```text
Initial static shop for plaskospb.ru v00.01 test 260707
```

4. Нажать **Commit changes**.

После этого сайт уже лежит в GitHub. Командная строка не нужна.

## 4. Подключить Cloudflare Pages

1. Открыть Cloudflare.
2. Перейти в **Workers & Pages**.
3. Нажать **Create application**.
4. Выбрать **Pages**.
5. Выбрать вариант импорта существующего Git-репозитория.
6. Подключить GitHub, если Cloudflare ещё не подключён к аккаунту.
7. Выбрать репозиторий `plaskospb-shop`.
8. Настройки сборки:
   - Framework preset: `None` / `Static HTML` / без фреймворка;
   - Build command: пусто;
   - Build output directory: `/`;
   - Root directory: `/`.
9. Нажать **Deploy**.

Cloudflare выдаст тестовый адрес вида:

```text
https://plaskospb-shop.pages.dev
```

## 5. Как обновлять сайт потом

Для мелкой правки отдельного файла:

1. Открыть файл на GitHub.
2. Нажать значок редактирования.
3. Внести правку.
4. Нажать **Commit changes**.
5. Cloudflare сам пересоберёт сайт.

Для замены версии целиком:

1. Открыть репозиторий.
2. Нажать **Add file → Upload files**.
3. Перетащить новые файлы поверх старых.
4. Нажать **Commit changes**.
5. Cloudflare сам сделает новый деплой.

## 6. Подключить домен plaskospb.ru

После проверки тестового адреса открыть проект в Cloudflare Pages:

1. Перейти в **Custom domains**.
2. Добавить:

```text
plaskospb.ru
www.plaskospb.ru
```

Если DNS домена уже в Cloudflare, записи обычно создаются автоматически. Если DNS не в Cloudflare, нужно будет следовать подсказкам Cloudflare по DNS-записям.

## 7. Что заменить до публичной публикации

В текущей тестовой версии есть заглушки. Перед публичным запуском заменить:

- телефон;
- email;
- цены;
- наличие;
- условия доставки;
- условия оплаты;
- юридические реквизиты;
- реальные товарные позиции;
- политику конфиденциальности и оферту, если магазин будет принимать заказы от физлиц.

Основные настройки сайта находятся в `script.js` и `data/products.json`.

## 8. Проверка после публикации

Проверить:

- открывается главная страница;
- работает каталог;
- работают фильтры;
- товар добавляется в корзину;
- количество в корзине меняется;
- заявка формируется корректно;
- кнопки связи ведут на правильный email/телефон;
- мобильная версия не ломается;
- `/robots.txt` открывается;
- `/sitemap.xml` открывается.

## 9. Когда нужна командная строка

Командная строка нужна только в более сложных случаях:

- когда сайт часто обновляется разработчиком;
- когда нужна нормальная история веток и откатов;
- когда используется сборка через Node/Vite/React и т.п.;
- когда нужно массово обновлять файлы скриптом.

Для текущей версии PLASKO SPB она не нужна.
